At the HiRo Lab I'm building Iona — a dual-arm mobile manipulator — into an end-to-end VLA system that takes natural-language commands and turns them into validated, executable robot actions.

The intelligence layer is a multi-agent LLM pipeline: four cooperating agents handling intent resolution, dialog, planning, and verification. The verification agent evaluates generated plans against parsed intent before execution — catching errors at planning time rather than at runtime — reaching 99.2% grounding accuracy on a 120-case automated test suite.

The robotics integration layer ties it to hardware: a fine-tuned YOLOv8 perception stack feeds MoveIt collision-aware pick/place/drop services, with ArUco-based frame calibration and a containerized ROS 2 ↔ ROS 1 bridge holding strict latency budgets across networked machines.

Battle-tested across eight consecutive live hardware demonstrations.

stack · task + motion planning oct – dec 2025

Task and motion planning for robotic manipulation

A complete TAMP pipeline fusing symbolic task planning (PDDL) with sampling-based motion planning (OMPL) for autonomous block manipulation in the Genesis simulator — with an execution-replanning loop that recovers from object slipping and tower collapse.

Built a hybrid AI system combining logical reasoning over PDDL predicates with continuous geometric planning for collision-free trajectories. The symbolic layer uses Pyperplan to decompose goals into action primitives; the motion layer uses OMPL sampling-based planners to realize each action in the workspace.

Benchmarked seven planners — RRT, RRT*, RRTConnect, PRM, EST, FMT, and BIT* — evaluating trade-offs between planning time, path optimality, and success rate across varying initial configurations.

The execution-replanning loop handles real-world failure modes: when blocks slip or towers collapse mid-task, the system automatically re-grounds symbolic predicates from perception and re-plans dynamically, enabling robust multi-block stacking under physical uncertainty.

stack · state estimation apr – may 2025

Deep & classical visual-inertial odometry

A stereo VIO system combining S-MSCKF filter-based fusion with a head-to-head comparison against three trained CNN-LSTM fusion networks — quantifying where classical filtering still beats learned approaches and where it doesn't.

Built a real-time S-MSCKF stereo VIO in C++ fusing IMU propagation with feature tracking, time alignment, and statistical outlier rejection. Achieves 0.091 m RMSE on the EuRoC benchmark for 6-DoF state estimation across multiple MAV sequences.

Trained and benchmarked three CNN-LSTM sensor-fusion networks against the classical filter in PyTorch on synthetic camera–IMU data — late fusion winning at 2.38 RMSE vs 5.24 / 2.82 for the other two architectures — with a reproducible, automated evaluation harness quantifying accuracy trade-offs.

The result is a clean comparison: filter-based methods remain the strongest baseline for accuracy on well-calibrated stereo+IMU, while learned approaches show promise specifically in fusion-strategy choice (late > early > intermediate in this setting).

05 / 08
stack · control oct – dec 2025

Quadrotor trajectory tracking via cascaded control

A real-time cascaded flight controller for a Crazyflie 2.0 — nested PID with outer position/velocity and inner attitude/rate loops — tuned for stable closed-loop 3D trajectory tracking with axis-by-axis error analysis.

Implemented a real-time cascaded flight controller for a Crazyflie 2.0 quadrotor with two nested loops: an outer position/velocity controller producing attitude setpoints, and an inner attitude/rate controller producing motor commands. Tuned the PID gain stack across both loops for stable closed-loop 3D trajectory tracking.

Achieved ~0.132 m 3D tracking RMSE and ~0.04 m horizontal RMSE across reference trajectories — the gap between the two metrics being the diagnostic itself.

Analyzed closed-loop error axis by axis to pinpoint altitude as the dominant error source, then systematically guided gain-tuning trade-offs to balance vertical stiffness against horizontal compliance. The methodology — let the per-axis residuals drive the next tuning step — generalizes beyond this airframe.

Implemented sampling-based and asymptotically optimal planners from scratch in C++ within the OMPL framework — applying core data structures (k-d trees, nearest-neighbor queries), graph search, and custom optimization objectives.

Wrote geometric collision-checking for point and rotating SE(2) robots, with configurable goal-biasing and parent-pointer tracking for solution-path reconstruction. The collision checker is reusable across planner variants.
